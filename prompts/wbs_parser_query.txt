WBS 텍스트:
{wbs_text}

## 파싱 요구사항

### 1. 기본 정보 추출
- 작업 ID (A, B, C, ...)
- 작업명 (한국어)
- 소요기간 (일 단위)
- 작업 유형 (EARTHWORK, CONCRETE, CRANE, STEEL, ELECTRICAL, PLUMBING, FINISHING)

### 2. 선행작업 관계 파싱
- 선행작업 ID
- 관계 유형 (FS, SS, FF, SF)
- 지연시간 (Lag)

### 3. 파싱 규칙
- "선행 없음" → predecessors: []
- "선행 A(FS)" → predecessors: [{"id": "A", "type": "FS", "lag": 0}]
- "선행 A(SS+1)" → predecessors: [{"id": "A", "type": "SS", "lag": 1}]
- "선행 A(FS), B(FS)" → predecessors: [{"id": "A", "type": "FS", "lag": 0}, {"id": "B", "type": "FS", "lag": 0}]

### 4. 작업 유형 추론
텍스트에서 명시되지 않은 경우 작업명에서 추론:
- "토공", "굴착" → EARTHWORK
- "콘크리트", "타설" → CONCRETE
- "크레인", "타워크레인" → CRANE
- "철골", "강재" → STEEL
- "전기", "배선" → ELECTRICAL
- "배관", "상하수도" → PLUMBING
- "마감", "마무리" → FINISHING

파싱 결과를 **다른 설명 없이, JSON 배열만** 제공해주세요.
